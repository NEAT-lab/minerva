# Minerva: 雙語會議輔助系統

這是一個基於物聯網架構的雙語會議輔助系統。系統能夠在會議中擷取語音、自動轉錄文字 (STT)、即時雙語翻譯、提供文法潤飾，並透過 LLM 分析對話上下文，即時給予與會者建議回覆。

系統取名為 *Minerva*，靈感汲取自羅馬神話中同時掌管智慧與編織的女神。這個名字完美契合了本專案的雙重核心價值：**智慧**代表著背後強大的大語言模型 (LLM) 與即時語意分析能力；而**編織**則象徵著系統透過物聯網架構，無縫串接各個邊緣節點的靈活擴展性。我們期許 Minerva 宛如會議中隱形的智者，不僅能將跨越語言與空間的對話緊密**編織**成無礙的互動，更能在無形之中，為每位與會者**編織**出最合適的建議回覆。

## 核心功能

- **即時語音轉文字（STT）**：透過 OpenAI Whisper 模型進行語音辨識。
- **即時雙語翻譯**：使用 Facebook NLLB-200 進行翻譯，並支援繁體中文。
- **即時語意分析與潤飾**：串接 Ollama (Llama 3)，提供發言者的即時英文文法潤飾，以及給予聆聽者建議回覆。
- **語音活動偵測（VAD）**：邊緣麥克風節點支援自動偵測說話與靜默，無須手動控制錄音開關。
- **動態資源分配**：支援設備註冊與心跳偵測，可透過會議管理介面將**麥克風**與**LLM 節點**動態分配至不同會議室。

## 系統架構概述

專案由五個主要模組構成，各節點與服務之間主要透過 MQTT 與 HTTP 協定進行資料通訊：
1. **MQTT Broker**
   - **功能**：訊息代理伺服器。
   - **說明**：負責轉發物聯網節點的訊息與指令。

2. **Registration Server** 
   - **功能**：物聯網節點註冊中心。
   - **說明**：接收所有裝置的 Description JSON，將設備資訊存入 SQLite 資料庫，並自動清理超時離線的設備。

3. **Function Server**
   - **功能**：會議管理伺服器。
   - **說明**：提供 API 建立與刪除會議室，並透過 MQTT 發送配對指令，將裝置配對至指定會議室中。

4. **Compute Node**
   - **功能**：語言模型運算。
   - **說明**：聆聽會議室音訊，使用 Whisper 進行語音轉文字、Ollama 進行語意推論、NLLB 進行翻譯，並將結果傳送給 Edge Node。

5. **Edge Node**
   - **功能**：邊緣端麥克風/播放器（樹莓派）。
   - **說明**：負責採集音訊、VAD 判斷斷句，並在樹莓派終端機渲染雙語字幕與建議回覆。
